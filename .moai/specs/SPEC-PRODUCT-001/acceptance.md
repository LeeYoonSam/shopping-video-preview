# 인수 기준: SPEC-PRODUCT-001 상품 리스트 페이지

## 1. 개요

본 문서는 SPEC-PRODUCT-001(상품 리스트 페이지)의 인수 기준과 테스트 시나리오를 정의합니다. Given-When-Then 형식으로 작성되었으며, 기능적 요구사항과 비기능적 요구사항을 모두 포함합니다.

---

## 2. 핵심 기능 시나리오

### 시나리오 1: 상품 리스트 정상 표시

```gherkin
Feature: 상품 리스트 표시
  사용자가 상품을 탐색할 수 있도록 상품 목록을 표시한다

  Scenario: 상품 리스트 페이지 정상 로드
    Given 사용자가 쇼핑몰에 접속한 상태이다
    When 사용자가 "/products" 페이지로 이동한다
    Then 20개의 상품 카드가 화면에 표시되어야 한다
    And 각 상품 카드에는 썸네일 이미지가 표시되어야 한다
    And 각 상품 카드에는 상품명이 표시되어야 한다
    And 각 상품 카드에는 가격이 원화(KRW) 형식으로 표시되어야 한다
```

### 시나리오 2: 반응형 그리드 레이아웃

```gherkin
Feature: 반응형 그리드 레이아웃
  다양한 화면 크기에서 최적의 그리드 레이아웃을 제공한다

  Scenario: 모바일 화면에서의 그리드
    Given 사용자가 320px ~ 767px 너비의 디바이스를 사용 중이다
    When 상품 리스트 페이지가 로드된다
    Then 상품 카드가 2열 그리드로 표시되어야 한다

  Scenario: 태블릿 화면에서의 그리드
    Given 사용자가 768px ~ 1279px 너비의 디바이스를 사용 중이다
    When 상품 리스트 페이지가 로드된다
    Then 상품 카드가 3열 그리드로 표시되어야 한다

  Scenario: 데스크톱 화면에서의 그리드
    Given 사용자가 1280px 이상 너비의 디바이스를 사용 중이다
    When 상품 리스트 페이지가 로드된다
    Then 상품 카드가 4열 그리드로 표시되어야 한다
```

### 시나리오 3: 상품 카드 호버 인터랙션

```gherkin
Feature: 상품 카드 인터랙션
  사용자에게 시각적 피드백을 제공한다

  Scenario: 상품 카드 호버 효과
    Given 사용자가 상품 리스트 페이지에 있다
    And 20개의 상품 카드가 표시되어 있다
    When 사용자가 특정 상품 카드에 마우스를 올린다
    Then 해당 카드에 미묘한 그림자 효과가 적용되어야 한다
    And 카드가 살짝 떠오르는 듯한 시각적 효과가 나타나야 한다
```

## 3. 엣지 케이스 시나리오

### 시나리오 4: 빈 상품 리스트

```gherkin
Feature: 빈 상태 처리
  상품이 없는 경우 적절한 안내를 제공한다

  Scenario: 상품 데이터가 없는 경우
    Given 상품 데이터가 빈 배열인 상태이다
    When 상품 리스트 페이지가 로드된다
    Then "상품이 없습니다" 메시지가 화면 중앙에 표시되어야 한다
    And 상품 카드 그리드는 표시되지 않아야 한다
```

### 시나리오 5: 이미지 로딩 실패

```gherkin
Feature: 이미지 오류 처리
  이미지 로드 실패 시 대체 이미지를 표시한다

  Scenario: 상품 이미지 로드 실패
    Given 상품 리스트 페이지가 로드된 상태이다
    And 특정 상품의 이미지 URL이 유효하지 않다
    When 해당 이미지 로드가 실패한다
    Then 기본 placeholder 이미지가 표시되어야 한다
    And 사용자에게 오류 메시지가 표시되지 않아야 한다
```

### 시나리오 6: 로딩 상태

```gherkin
Feature: 로딩 상태 표시
  데이터 로딩 중 스켈레톤 UI를 표시한다

  Scenario: 페이지 초기 로딩
    Given 사용자가 상품 리스트 페이지로 이동 중이다
    When 데이터 로딩이 진행 중이다
    Then 스켈레톤 카드 그리드가 표시되어야 한다
    And 스켈레톤은 실제 카드와 동일한 그리드 레이아웃을 가져야 한다
```

### 시나리오 7: 유효하지 않은 가격 데이터

```gherkin
Feature: 가격 데이터 유효성 처리
  유효하지 않은 가격에 대한 대체 표시를 제공한다

  Scenario: 가격이 null 또는 undefined인 경우
    Given 상품 데이터 중 가격이 null인 상품이 있다
    When 해당 상품 카드가 렌더링된다
    Then 가격 영역에 "가격 정보 없음"이 표시되어야 한다

  Scenario: 가격이 음수인 경우
    Given 상품 데이터 중 가격이 음수인 상품이 있다
    When 해당 상품 카드가 렌더링된다
    Then 가격 영역에 "가격 정보 없음"이 표시되어야 한다
```

## 4. 성능 기준

### 4.1 로딩 성능

| 지표 | 기준값 | 측정 방법 |
|------|--------|-----------|
| First Contentful Paint (FCP) | 1.5초 이하 | Lighthouse |
| Largest Contentful Paint (LCP) | 2.5초 이하 | Lighthouse |
| Cumulative Layout Shift (CLS) | 0.1 이하 | Lighthouse |
| Time to Interactive (TTI) | 3.5초 이하 | Lighthouse |

### 4.2 이미지 최적화

```gherkin
Feature: 이미지 성능 최적화
  이미지 로딩이 페이지 성능에 미치는 영향을 최소화한다

  Scenario: 이미지 Lazy Loading
    Given 상품 리스트 페이지가 로드된다
    When 페이지 하단의 상품 카드가 뷰포트 밖에 있다
    Then 해당 이미지는 즉시 로드되지 않아야 한다
    And 스크롤하여 뷰포트에 진입할 때 로드되어야 한다

  Scenario: 이미지 포맷 최적화
    Given 상품 이미지가 요청된다
    When 브라우저가 WebP를 지원한다
    Then WebP 포맷으로 이미지가 제공되어야 한다
```

## 5. 접근성 기준

### 5.1 WCAG 2.1 AA 준수 사항

| 기준 | 요구사항 | 검증 방법 |
|------|----------|-----------|
| 1.1.1 Non-text Content | 모든 이미지에 대체 텍스트 제공 | alt 속성 확인 |
| 1.4.3 Contrast | 텍스트 명암비 4.5:1 이상 | 색상 대비 검사 도구 |
| 2.1.1 Keyboard | 키보드로 모든 기능 접근 가능 | 키보드 탐색 테스트 |
| 2.4.4 Link Purpose | 링크 목적 명확히 식별 가능 | 스크린 리더 테스트 |

### 5.2 접근성 시나리오

```gherkin
Feature: 스크린 리더 지원
  시각 장애 사용자가 상품 정보를 이해할 수 있다

  Scenario: 상품 이미지 대체 텍스트
    Given 스크린 리더 사용자가 상품 리스트를 탐색 중이다
    When 상품 카드에 포커스가 이동한다
    Then 상품 이미지의 alt 텍스트가 읽혀야 한다
    And alt 텍스트에 상품명이 포함되어야 한다

  Scenario: 가격 정보 접근성
    Given 스크린 리더 사용자가 상품 카드를 탐색 중이다
    When 가격 정보에 도달한다
    Then 가격이 원화 단위로 명확하게 읽혀야 한다
    And 할인 상품의 경우 원가와 할인가가 구분되어 읽혀야 한다
```

```gherkin
Feature: 키보드 접근성
  키보드만으로 모든 기능에 접근할 수 있다

  Scenario: 키보드 탐색
    Given 사용자가 키보드만 사용 중이다
    When Tab 키를 눌러 상품 카드를 탐색한다
    Then 각 상품 카드에 순차적으로 포커스가 이동해야 한다
    And 포커스된 카드에 시각적 표시가 있어야 한다
```

## 6. 디자인 검증

### 6.1 시각적 일관성

| 검증 항목 | 기준 | 확인 방법 |
|-----------|------|-----------|
| 색상 팔레트 | 흑백 모노크롬 | 시각적 검사 |
| 타이포그래피 | 지정된 크기/굵기 | DevTools 확인 |
| 간격 | 지정된 패딩/마진 | DevTools 확인 |
| 그리드 정렬 | 균등 배치 | 시각적 검사 |

### 6.2 디자인 시나리오

```gherkin
Feature: 모노크롬 디자인 적용
  모던하고 미니멀한 흑백 디자인을 유지한다

  Scenario: 색상 팔레트 검증
    Given 상품 리스트 페이지가 로드된다
    When 페이지의 색상을 검사한다
    Then 배경색은 #FFFFFF(흰색)이어야 한다
    And 주 텍스트 색상은 #000000(검정)이어야 한다
    And 보조 텍스트 색상은 #666666(회색)이어야 한다

  Scenario: 호버 상태 색상
    Given 상품 카드가 표시된 상태이다
    When 마우스를 카드 위에 올린다
    Then 카드 배경이 #F5F5F5(연한 회색)으로 변경되어야 한다
```
